<decisions_made>

- Photorelays over CD4066/MOSFETs: galvanic isolation prevents noise coupling on sensitive analog button lines. The lines already showed phantom presses from an unterminated cable. Isolation eliminates shared ground as a noise path.
- AQY212EH (Panasonic, DIP-4): 25 ohm on-resistance, 1uA max off-leakage, breadboard-friendly. 10-pack ordered for spares.
- B0505S-1W isolated DC-DC: SIP through-hole package, breadboard compatible. Allows tapping board +5V (Pin 1) while maintaining isolation. Not needed for v1 (USB battery), but future-proofs the design.
- RJ45 screw terminal breakout (2x female, straight-through): replaces T-splitter + cut cable + Wago setup. Cleaner tap access to all 8 pins.
- ESP32 WROOM-32 DevKitC (USB-C, CH340C, 2-pack): $0.99 for two boards. Powers from VIN pin with onboard 3.3V regulator.
- "Re-home" pattern for open-loop control: slam temp down 25 times to guaranteed floor (80F), then count up to target. Eliminates cumulative drift from missed presses, manual adjustments, or power cycles.
- Phase 2 (display reading) is required, not optional: open-loop control is too fragile for unattended long-term use. MagnusPer/Balboa-GS510SZ project is closest reference for the synchronous clock+data protocol.
- Phase 3: community contribution -- publish VS-series protocol documentation, circuit design, and working firmware. No published work exists for this board family.
</decisions_made>

<context>
The VS300FL4 is a Balboa hot tub controller with a "dumb" VL-series topside panel. The panel has no microcontroller -- buttons are analog voltage switches (idle ~2.3V, pressed ~4.7V via bridge to +5V). Two RS-485 channels (Pin 5, Pin 6) carry display data board-to-panel only. Manual button simulation confirmed working (bridging Pin 1 to Pin 8 lowered setpoint by 1F).

The user has existing hardware from Phase 1 protocol verification: Monoprice RJ45 T-splitter, Waveshare USB RS-485 adapter, sacrificial Cat5 + Wago breakout, Lonely Binary 8ch logic analyzer. Also has 5x MAX485 boards on hand.

Key reference: reference/status_03_08.md contains full pinout, protocol captures, and voltage measurements.

User preferences: uses Python with uv, wants tests before features, no emojis, no Co-Authored-By in commits. This is a hardware+firmware project so testing approach will need adaptation (can't unit test analog circuits).

The user explicitly wants to contribute findings back to the community (GitHub) if the project works. This should be a milestone.

Circuit per button:
  ESP32 GPIO -> 680R -> AQY212EH LED+ / LED- -> ESP32 GND  (isolated)
  AQY212EH OUT1 -> +5V (Pin 1) / OUT2 -> Button Pin        (hot tub side)

Board's RJ45 pinout:
  Pin 1: +5V | Pin 2: Temp Up | Pin 3: Lights | Pin 4: GND
  Pin 5: RS-485 display content | Pin 6: RS-485 display refresh
  Pin 7: Jets | Pin 8: Temp Down
</context>
